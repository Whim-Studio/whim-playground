# TIWAS RPG — Shared Interface Contract (binding for Tasks 1, 2, 3)

A standalone **Java 8 Swing** desktop app implementing the Tiwas RPG: a character
creator and a modular adventure loader, with strict JSON save/load. **Zero external
libraries** — only `javax.swing`, `java.awt`, `java.util`, `java.io`, `java.lang`.
No `var`, no text blocks, no switch expressions, no `java.time`/streams-only APIs that
postdate Java 8 is fine to use (streams exist in 8) but keep it simple and Java-8 legal.

This document is the **single source of truth**. Every class name, package, method
signature, and field below is a hard contract. Do not rename or restructure shared
types. If something is underspecified, pick the simplest Java-8 implementation and keep
the signatures exactly as written.

## Task 1 — Domain & JSON (package `com.tiwas.rpg.domain`, `com.tiwas.rpg.json`)

### `AttributeCode` enum — the 24 attributes, body group first then mind group

Each constant carries: `code` (3-letter lowercase), `animal`, `fullName`,
`tier1Skill`, `body` (boolean). Declare in EXACTLY this order:

```
Body:  BPP Bear "Body Power Power" Might | BPS Tiger "Body Power Speed" Impact |
       BPE Elephant "Body Power Endurance" Brawn | BPX Dragon "Body Power Social" Presence |
       BSP Hawk "Body Speed Power" Agility | BSS Rat "Body Speed Speed" Reflexes |
       BSE Horse "Body Speed Endurance" Quickness | BSX Monkey "Body Speed Social" Grace |
       BEP Badger "Body Endurance Power" Toughness | BES Wolf "Body Endurance Speed" Stamina |
       BEE Ox "Body Endurance Endurance" Vitality | BEX Dog "Body Endurance Social" Poise
Mind:  MPP Stag "Mind Power Power" Cunning | MPS Snake "Mind Power Speed" Wits |
       MPE Rooster "Mind Power Endurance" Willpower | MPX Fox "Mind Power Social" Glamour |
       MSP Bat "Mind Speed Power" Acuity | MSS Cat "Mind Speed Speed" Perception |
       MSE Rabbit "Mind Speed Endurance" Alacrity | MSX Otter "Mind Speed Social" Charm |
       MEP Crane "Mind Endurance Power" Focus | MES Goat "Mind Endurance Speed" Discipline |
       MEE Owl "Mind Endurance Endurance" Resolve | MEX Pig "Mind Endurance Social" Composure
```

Required API:
```java

public String animal();          // "Bear"
public String fullName();        // "Body Power Power"
```

### `Character`

```java
public final class Character {
    public Character();
    public Character(String name);

    public String getName();
    public void setName(String name);

    // 24 attributes, stored in an EnumMap<AttributeCode,Integer>. Missing = 0.
    public int getAttribute(AttributeCode a);

    public java.util.Map<String,Skill> getSkills();
    public void putSkill(Skill s);
    public Skill getSkill(String name);

    // Active resource pools (current values; max values are derived getters below).
    public int getCurrentHP();   public void setCurrentHP(int v);
    public int getCurrentPE();   public void setCurrentPE(int v);
    public int getCurrentMP();   public void setCurrentMP(int v);
    public int getGeneralXP();   public void setGeneralXP(int v);

    public java.util.List<String> getInventory(); // simple item-name list

    // Derived stats — sums of attribute groups, round down. Pure functions of attributes.
    public int getMaxHP();            // sum of all 12 body attributes
    public int getMaxMP();            // sum of all 12 mind attributes
    public int getMaxPhysicalEnergy();// bep + bes + bee
    public int getSpeed();            // bsp + bss + bse
    public int getEnergyRegen();      // bep + bes
    public int getMpRegen();          // mep + mes
    public int getMovementSpeed();    // (bsp + bss) / 15

    // Sets current pools to their maxima (PE = max physical energy, MP = max MP, HP = max HP).
    public void restoreToFull();

    // JSON
    public java.util.Map<String,Object> toMap();
    public static Character fromMap(java.util.Map<String,Object> m);
    public String toJson();                              // pretty
    public static Character fromJson(String json);
    public void save(java.io.File f) throws java.io.IOException;
    public static Character load(java.io.File f) throws java.io.IOException;
}
```

> Note: "current PE" tracks the Physical-Energy pool used for Body skill rolls
> (max = `getMaxPhysicalEnergy()`). "current MP" tracks the Mind pool
> (max = `getMaxMP()`). "current HP" max = `getMaxHP()`. `restoreToFull()` sets
> currentPE=maxPhysicalEnergy, currentMP=maxMP, currentHP=maxHP.

### `com.tiwas.rpg.json.Json` — generic parser/writer (hand-written, no libraries)

Parses to a tree of `java.util.Map<String,Object>` (object, insertion-ordered via
`LinkedHashMap`), `java.util.List<Object>` (array), `String`, `Double` (all numbers),
`Boolean`, and `null`. Writer accepts the same tree.

```java
public final class Json {
    public static Object parse(String text);          // throws JsonException on malformed input
    public static String write(Object value);         // compact
    public static String writePretty(Object value);   // 2-space indented

    // Convenience coercions (so domain mappers stay terse):
    public static java.util.Map<String,Object> asObject(Object o);
    public static java.util.List<Object> asArray(Object o);
    public static String asString(Object o);
    public static int asInt(Object o);                // truncates Double toward zero
    public static boolean asBoolean(Object o);
}
public class JsonException extends RuntimeException { public JsonException(String msg); }
```

Requirements: handle nested objects/arrays, strings with escapes (`\" \\ \/ \n \t \r \b \f \uXXXX`),
integers and decimals, negative numbers, booleans, null, and arbitrary whitespace.
Round-trip safety: `Json.parse(Json.write(x))` must reproduce an equal tree.

### `ActionResolver` — the core loop (roll-under, exertion, overflow, margin, XP, doubles, recovery)

```java
public final class ActionResolver {
    public ActionResolver(Dice dice);

    // Resolve a single skill test for `actor` using `skill` at difficulty modifier `dm`.
    // Steps (all rounding DOWN):
    //   effectiveSkill = clamp(skill.getValue() + dm, 1, 99)
    //   roll = dice.d100()
    //   success = roll <= effectiveSkill
    //   pool = skill.isMind() ? currentMP : currentPE
    //   cost = roll
    //   if cost <= pool: pool -= cost; overflow = 0
    //   else: overflow = cost - pool; pool = 0; currentHP -= overflow   (HP clamps at 0; mark dead if <=0)
    //   margin = success ? (effectiveSkill - roll) / 10 : 0
    //   doubles = roll in {11,22,33,44,55,66,77,88,99}
    //   failureXP = success ? 0 : (roll - effectiveSkill)              (min 0)
    //   unlockedAdvancedSkill = (!success && doubles)
    //   recovered = (skill.isMind() ? mpRegen : energyRegen) / 2; pool = min(pool + recovered, poolMax)
    //   write back currentMP/currentPE and currentHP onto actor.
    public ActionResult resolve(Character actor, Skill skill, int dm);
    public ActionResult resolve(Character actor, Skill skill); // dm = 0
}
```

### `ActionResult` (engine carrier — immutable, public getters)
Fields/getters: `int roll`, `int effectiveSkill`, `boolean success`, `int cost`,
`int overflowDamage`, `int margin`, `boolean doubles`, `int failureXP`,
`boolean unlockedAdvancedSkill`, `int recovered`, `boolean usedMind`,
`int newPoolValue`, `int newHP`, and `String describe()` returning a one-line
human-readable summary (e.g. `"Might: rolled 27 ≤ 40 → SUCCESS, paid 27 PE, margin +1"`).

## Task 3 — Swing UI (package `com.tiwas.rpg.ui`, `com.tiwas.rpg.app`, imports `domain` + `engine`)

`com.tiwas.rpg.app.Main`: set system look-and-feel, build the `MainFrame` on the EDT
(`SwingUtilities.invokeLater`), title `"Tiwas RPG — Demo Version"`. Show "Demo Version"
prominently somewhere in the window.

`MainFrame` (JFrame): a `JTabbedPane` (or CardLayout) with two views:

1. **Character Creator** (`CharacterCreatorPanel`):
   - A name `JTextField` and a **"Roll Character"** button. On click, call
     `new CharacterGenerator(new Dice()).generate(name)` and display:
     - The 24 attributes in a clean, scannable grid, grouped Body (12) then Mind (12),
       each row showing animal, full name, Tier-1 skill, and the rolled value.
     - A derived-stats panel: Max HP, Max MP, Physical Energy, Speed, Energy Regen,
       MP Regen, Movement Speed — read from the `Character` getters.
   - **"Save JSON"** → `JFileChooser` → `character.save(file)`.
   - **"Load JSON"** → `JFileChooser` → `Character.load(file)` → re-render the grid.

2. **Adventure Loader / Session** (`AdventurePanel`):
   - **"Load Adventure JSON"** → `JFileChooser` → `AdventureModule.load(file)`; show
     title, author, description, and the list of scenes (and NPCs).
   - **"Select Character JSON"** → load a saved `Character`.
   - **"Begin Session"** → an active panel showing the character's live pools
     (HP/PE/MP with current vs max). The user picks one of the character's skills
     from a `JComboBox`, optionally sets a Difficulty Modifier (spinner, e.g. -50..+30),
     clicks **"Attempt Action"**, which calls `ActionResolver.resolve(character, skill, dm)`
     and appends `result.describe()` to a log `JTextArea`, then refreshes the pool
     displays (showing overflow damage to HP when it happens).
   - **"Save JSON" / "Load JSON"** for the character within the session too.

Keep the UI responsive: never block the EDT on file IO beyond a `JFileChooser`
(small files, synchronous is fine). Use `GridBagLayout`/`BorderLayout`/`JTable` or
`JPanel` grids — your call — but the attribute grid must be readable at a glance.

UI talks to the engine only through `CharacterGenerator`, `ActionResolver`, `Dice`,
and to domain through the public getters/`save`/`load`/`fromJson` methods above.
